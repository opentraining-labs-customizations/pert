= Quick Course for OCP

== Overview

* Deploys Quick Course content to OpenShift using Helm
* Based on the same showroom-deployer helm chart infrastructure
* Optimized for Red Hat Quick Course static documentation sites
* Supports single or multiple users (autodetected)
* Optional terminal support (wetty/showroom) - defaults to content-only deployment
* Builds Quick Course content using npm build (Antora via Node.js)

== Key Differences from Showroom

* **Content Type**: Quick Course templates use pre-built static sites (npm build → build/site/)
* **Default Mode**: Content-only (no terminals) - most quick courses are pure documentation
* **Variable Injection**: Optional - quick courses typically don't need dynamic variables
* **Build System**: Uses npm/Node.js build process with pre-built UI bundles

== Usage

.Integrating Quick Course into your `config/env_type`:
* Add the `ocp4_workload_quickcourse` infra role to your deployment
* Configure the content repository and build settings
* Optional: Enable terminals if your course requires SSH access

== Basic Configuration

[source,yaml]
----
infra_workloads:
[ ... all your other infra roles ... ]
- ocp4_workload_quickcourse
----

== Which Quick Course repo to deploy?

[source,yaml]
----
ocp4_workload_quickcourse_content_git_repo: "https://github.com/RedHatQuickCourses/aap-on-openshift.git"
# Optional: tag, branch, etc. Default is main.
ocp4_workload_quickcourse_content_git_repo_ref: main
----

== Selecting the Helm Chart

Quick Course uses the same showroom-deployer Helm Chart infrastructure:
https://github.com/rhpds/showroom-deployer/charts/showroom-single-pod

.Example
[source,yaml]
----
# Chart package URL (default):
ocp4_workload_quickcourse_chart_package_url: https://rhpds.github.io/showroom-deployer

# Chart name (default):
ocp4_workload_quickcourse_deployer_chart_name: showroom-single-pod

# Pick the latest chart available from https://github.com/rhpds/showroom-deployer/releases
ocp4_workload_quickcourse_deployer_chart_version: "1.3.4"
----

== Deployment Type

The default deployment type is `helm` (deployed via Helm CLI on bastion).

.Example
[source,yaml]
----
# How to deploy Quick Course. Options are `helm`
ocp4_workload_quickcourse_deployment_type: helm
----

== Content-Only Deployment (Default)

Most Quick Courses are pure documentation and don't need terminals. This is the default mode.

.Example
[source,yaml]
----
ocp4_workload_quickcourse_content_git_repo: "https://github.com/RedHatQuickCourses/aap-on-openshift.git"
ocp4_workload_quickcourse_content_git_repo_ref: main
ocp4_workload_quickcourse_content_only: true  # Default
----

== Enabling Terminal Support (Optional)

If your Quick Course requires SSH access to a bastion or terminal interaction:

=== Wetty Terminal with Auto-SSH to Bastion

.Example
[source,yaml]
----
ocp4_workload_quickcourse_content_only: false
ocp4_workload_quickcourse_terminal_type: wetty
ocp4_workload_quickcourse_wetty_ssh_bastion_login: true
----

=== Showroom Terminal (OpenShift CLI access)

.Example
[source,yaml]
----
ocp4_workload_quickcourse_content_only: false
ocp4_workload_quickcourse_terminal_type: showroom
----

=== Disable Terminals (Default)

.Example
[source,yaml]
----
ocp4_workload_quickcourse_content_only: true
# Or set terminal_type to empty string:
ocp4_workload_quickcourse_terminal_type: ""
----

== Variable Injection (Optional)

Quick Courses typically use static content, but you can inject variables into the Antora build if needed.

.Example
[source,yaml]
----
# Enable variable injection
ocp4_workload_quickcourse_var_inject: true

# Auto-collect all playbook variables
ocp4_workload_quickcourse_auto_collect_vars: true

# Or explicitly define variables
quickcourse_vars:
  openshift_version: "4.14"
  cluster_name: "my-cluster"
----

== Advanced Terminal Options

=== Stacked Terminals (Two Terminals in One Tab)

.Example
[source,yaml]
----
ocp4_workload_quickcourse_content_only: false
ocp4_workload_quickcourse_stacked_terminals_enable: true
ocp4_workload_quickcourse_stacked_terminals_display_name: "Terminals"
ocp4_workload_quickcourse_terminal_type: wetty  # or showroom
----

=== Second Terminal Tab

.Example
[source,yaml]
----
ocp4_workload_quickcourse_content_only: false
ocp4_workload_quickcourse_second_terminal_tab_enable: true
ocp4_workload_quickcourse_second_terminal_tab_display_name: "Terminal 2"
ocp4_workload_quickcourse_terminal_type: showroom
----

Control CPU and memory for terminal pods:

== Multi-User Support

The role automatically detects multi-user environments via agnosticd_user_data.
Each user gets their own namespace: `quickcourse-{{ guid }}-{{ username }}`

== Namespace Configuration

.Example
[source,yaml]
----
# Single user namespace
ocp4_workload_quickcourse_namespace: "quickcourse-{{ guid }}"

# Display name for the namespace
ocp4_workload_quickcourse_namespace_display: Quick Course Guide
----

== Custom Antora Playbook

If your Quick Course uses a non-standard Antora playbook name:

.Example
[source,yaml]
----
ocp4_workload_quickcourse_content_antora_playbook: custom-playbook.yml
----

== Relationship to quickcourse_deployment Role

* **quickcourse_deployment**: Bare metal deployment using podman + systemd + Traefik
* **ocp4_workload_quickcourse**: OpenShift deployment using Helm + Routes

Both roles deploy Quick Course content but target different environments. The bare metal version builds and serves content directly with npm/podman. The OpenShift version uses Helm charts to deploy to OpenShift clusters.

== Requirements

* OpenShift cluster with cluster-admin access
* kubectl/oc CLI installed
* Helm will be auto-installed if not present
* AgnosticD environment (for user data and multi-user support)

== Default Variables

See `defaults/main.yaml` for the complete list of configurable variables.

Key defaults:
* `ocp4_workload_quickcourse_content_only: false`
* `ocp4_workload_quickcourse_terminal_type: ""` (no terminal)
* `ocp4_workload_quickcourse_deployment_type: helm`
* `ocp4_workload_quickcourse_var_inject: true`
* `ocp4_workload_quickcourse_auto_collect_vars: true`

== Output

After successful deployment, the role outputs:
* `lab_ui_url`: Main Quick Course URL
* `quickcourse_primary_view_url`: Same as lab_ui_url

These are available via agnosticd_user_info and in `user-info.yaml`.
